02 Reduce Sum

From Correctness To Cooperation

Reduction should also be learned as a ladder. We start with the easiest correct version, then improve how threads cooperate.

Source: src/cuda/02_reduce_sum.cu

Step 0: What Makes Reduction Different?

many inputs
→
one output

Elementwise kernels are easy because threads do not need each other. Reduction is different: many threads want to contribute to one final value.

Step 1: Atomic Version

const int index = blockIdx.x * blockDim.x + threadIdx.x;
if (index < count) {
  atomicAdd(output, input[index]);
}
Why start here? It is easy to explain and obviously correct.
Why not stop here? Too many threads fight over one memory location.

Why Atomics Become A Bottleneck

thread 0
↘
same output
↗
thread 255
All threads are serialized around the same destination. So the next optimization idea is obvious: reduce locally first, then write fewer partial sums.

Step 2: Shared-Memory Tree Reduction

Tree pattern inside one block
256
→
128
→
64
→
32
→
1
shared[threadIdx.x] = input[global_index];
__syncthreads();

for (int offset = blockDim.x / 2; offset > 0; offset /= 2) {
  if (threadIdx.x < offset) {
    shared[threadIdx.x] += shared[threadIdx.x + offset];
  }
  __syncthreads();
}
Now each block produces one partial sum instead of every thread updating the final answer directly.

Why Tree Reduction Is Better

Shared memory is local Block-local communication is cheaper than global contention.
Traffic drops One block writes one partial result, not hundreds of atomic updates.
But We still pay for a lot of block-wide synchronization.

Step 3: Warp-Aware Reduction

Once the active values fit inside a warp, we no longer need full-block synchronization for every final step.

for (int offset = warpSize / 2; offset > 0; offset /= 2) {
  value += __shfl_down_sync(0xffffffff, value, offset);
}
This is the next layer of the ladder: move from block-wide cooperation to warp-level exchange.

Step 4: Hierarchical Chunking

one thread loads many elements
→
thread-local sum
→
block reduction

Before the shared-memory tree even begins, each thread can pre-accumulate a chunk of data. That reduces pressure on the later stages.

The Full Learning Ladder

Code Order

atomic -> shared-memory tree -> warp-aware -> hierarchical chunked

Question To Ask

At each step, where does the partial sum live: global memory, shared memory, warp registers, or thread-local registers?
