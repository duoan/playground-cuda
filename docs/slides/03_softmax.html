03 Softmax

From Stable Math To Attention-Ready Softmax

This deck follows the code ladder: stable CPU idea first, then naive GPU, then cooperative block softmax, then online and transformer-aware variants.

Softmax is not mysterious. It is two reductions plus one elementwise normalization.

Step 1: Stable Reference Version

float max_value = row_ptr[0];
for (int col = 1; col < cols; ++col) {
  max_value = fmaxf(max_value, row_ptr[col]);
}

float row_sum = 0.0f;
for (int col = 0; col < cols; ++col) {
  row_sum += expf(row_ptr[col] - max_value);
}
Subtracting the max is the first lesson. Without it, the math can overflow badly.

Step 2: Naive GPU Softmax

Idea One thread handles one row by itself.
Why good for learning It matches the CPU logic almost exactly.
Why limited One thread does too much serial work on long rows.
This is the same learning move as before: start with a direct mapping, then ask how to make more threads cooperate on one row.

Step 3: Block Cooperative Softmax

One row shared across a block
Different threads cover different columns of the same row.

Now the row becomes a cooperative reduction problem, much like reduce_sum.

Why Another Step Is Needed

Problem Standard softmax still assumes you can logically see the whole row together.
But attention wants more In tiled attention, scores arrive chunk by chunk.

That is why the next version is online softmax.

Step 4: Online Softmax

new_m = max(old_m, tile_max);
new_l = old_l * exp(old_m - new_m)
      + tile_l * exp(tile_max - new_m);
Online softmax is the bridge to FlashAttention. It lets you update softmax statistics tile by tile instead of needing the full row at once.

Now the kernel is no longer just “softmax.” It becomes the exact variant that attention workloads need.
