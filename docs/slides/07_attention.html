07 Attention

Where Earlier Kernels Meet

Attention combines GEMM-like score computation, row-wise softmax, and a weighted sum over values. It is the first operator in the repo that truly forces you to think about dataflow.

Source: src/cuda/07_attention.cu

Output Row
score += q[q_idx * head_dim + d] * k[k_idx * head_dim + d];
prob = expf(score - row_max) / row_sum;
out += prob * v[k_idx * head_dim + d];
This is literally GEMM-ish dot products, then softmax, then another weighted accumulation.

FlashAttention does not change the math. It changes how these tiles are visited so the score matrix never has to live in full in HBM.
What to map in your head
Q tile Queries are often kept fixed while keys stream past.
K tile Produces a small score tile with the current Q tile.
Softmax state Row max and row sum must be updated stably.
V tile Uses the normalized weights to accumulate output.
