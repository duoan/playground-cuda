01 Vector Add

From One Thread, One Element

This deck should be read exactly like the code file: start from the most naive version, feel its limits, then climb one optimization step at a time.

Source: src/cuda/01_vector_add.cu

Step 0: What Problem Are We Solving?

a[i]
+
b[i]
→
c[i]

Vector add is intentionally simple. That lets you focus on thread indexing and memory mapping instead of difficult math.

Step 1: Naive Kernel

blockIdx.x
x
blockDim.x
+
threadIdx.x
→
global index i
const int index = blockIdx.x * blockDim.x + threadIdx.x;
if (index < count) {
  c[index] = a[index] + b[index];
}
One thread owns one output element. This is the most important first mental model in CUDA.

Why Naive Is Not Enough

Problem 1 A thread only does one tiny unit of work.
Problem 2 If the launch is too small, some data may never be touched.
Problem 3 We are not yet thinking in tiles or chunks of data.
The code is correct. The next versions are about teaching better ways to assign larger amounts of work to each block and thread.

Step 2: Block Tile Thinking

One block owns a contiguous chunk
The highlighted region is one logical tile.
const int tile_start = blockIdx.x * kTileSize;
const int thread_base = tile_start + threadIdx.x;

for (int item = 0; item < kItemsPerThread; ++item) {
  const int index = thread_base + item * kBlockThreads;
  if (index < count) {
    c[index] = a[index] + b[index];
  }
}
One thread now computes multiple elements inside the block tile.

Step 3: Grid-Stride Loop

Now we solve a different problem: what if the array is much larger than the first launch covers?

for (int i = index; i < count; i += blockDim.x * gridDim.x) {
  c[i] = a[i] + b[i];
}
The same thread comes back and handles later elements, separated by one full-grid stride. This makes the kernel scale more gracefully.

Step 4: Vectorized Access

load float4
→
add 4 values
→
store float4

The math is unchanged. Only memory access becomes wider and more bandwidth-aware.

The Full Learning Ladder

Code Order

naive -> block-tiled -> grid-stride -> vectorized
